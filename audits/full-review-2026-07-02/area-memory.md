## Staleness risks

- **Memory model shape.** The guide's discrete key-value ledger matches 2024-era ChatGPT memory and Gemini Saved Info. By 2025–2026, ChatGPT moved to a two-tier system (saved memories + implicit chat-history insights that have no entry-level UI) and Claude launched prose-summary memory. Content written March–June 2026 that presents entry-level ledgers as *the* pattern is already fighting the field's direction; the guide never mentions the summary/implicit tier at all.
- **Cursor assumptions.** Any implicit reliance on Cursor's approval-based Memories flow (the closest real precedent for the Save/Dismiss banner) predates Cursor's current major version; the feature's consent flow and settings location have shifted across releases and should be re-verified against the current release, not 1.x-era behavior.
- **Privacy-toggle landscape.** Temporary/incognito modes (ChatGPT Temporary Chat, Claude incognito) have become the dominant "all off" mechanism — a *mode*, not a settings preset. The All Off preset framing may read dated.
- **"(§6.)" internal citation** in `principles-data.ts` will be meaningless to public readers if the underlying document isn't published alongside the guide.
- Not staleness but adjacent: the context ring's detail is hover/focus-only (`Tooltip`); the guide's own claim "details revealed on hover" has no stated touch behavior, unlike the citation pattern on the Sources page which explicitly specs tap/Escape.

# PRODUCT CHECK

## CONFIRMED

**Disable ≠ delete (access policy ≠ retention policy) — the guide's strongest asserted claim, verbatim true in two major products.**
- ChatGPT: "Turning saved memory off won't delete anything that's already been remembered. And deleting a chat doesn't remove saved memory from that conversation." — [OpenAI Memory FAQ](https://help.openai.com/en/articles/8590148-memory-faq) (surface: Settings → Personalization; evidence July 2026 search snapshot). The 2026 release notes add a combined "Delete and turn off memory" three-dot action, and note that re-enabling may create *new* memories from retained chat history — [ChatGPT Release Notes](https://help.openai.com/en/articles/6825453-chatgpt-release-notes).
- Microsoft 365 Copilot: turning off the Saved memories control "is stopped from applying saved memories to the chat, but doesn't remove saved memories" — [Microsoft Learn, copilot-personalization-memory](https://learn.microsoft.com/en-us/copilot/microsoft-365/copilot-personalization-memory) (doc dated 2025-11-18, updated 2026-05-22). **Nuance the guide should add:** for Copilot's *chat-history-derived* details, disabling DOES delete after 30 days — so the distinction holds for saved memories but not uniformly across memory tiers.

**Inline edit of individual memory entries exists in real products** (the audit feared it was stronger than reality — it isn't).
- Gemini "Saved info": add/edit/delete free-text entries at gemini.google.com/saved-info — [Google announcement coverage](https://craftium.ai/google-gemini-saved-info-feature/), [PopSci how-to](https://www.popsci.com/diy/how-to-erase-google-gemini-memory/) (Nov 2024 feature, still current 2025–26).
- Microsoft 365 Copilot: "you can view, edit, or delete your memories anytime from the Settings pane" — [Microsoft Community Hub, "Introducing Copilot Memory"](https://techcommunity.microsoft.com/blog/microsoft365copilotblog/introducing-copilot-memory-a-more-productive-and-personalized-ai-for-the-way-you/4432059) (GA July 2025).
- Devin: Knowledge entries fully editable, including editing a *suggested* entry before saving — [Devin docs, Knowledge](https://docs.devin.ai/product-guides/knowledge).
- Counterpoint to keep: ChatGPT's Manage memories panel is view/delete only (trash icon per entry, delete-all); edits are conversational — [OpenAI Memory FAQ](https://help.openai.com/en/articles/8590148-memory-faq), [How-To Geek](https://www.howtogeek.com/how-to-disable-chatgpt-memory-and-delete-saved-memory/).

**Save/Dismiss consent banner has a real, nameable precedent — in dev agents, not chat assistants.**
- Cursor: changelog 1.2 (July 3, 2025) — Memories GA "introduced user approvals for background-generated memories to preserve trust" — [cursor.com/changelog/1-2](https://cursor.com/changelog/1-2). Surface: in-editor approval prompt; memories managed in Settings → Rules, project-scoped ([docs](https://docs.cursor.com/en/context/memories), [forum](https://forum.cursor.com/t/rules-vs-memories-and-global-vs-project/137149)).
- Devin: "Devin will automatically suggest Knowledge to remember based on your feedback in chat. You can edit the suggested Knowledge before saving, or dismiss the Knowledge if it's not helpful," plus regenerate-on-feedback — [docs.devin.ai/product-guides/knowledge](https://docs.devin.ai/product-guides/knowledge). Note Devin's flow is richer than the guide's (edit-before-save and regenerate, not just Save/Dismiss).
- ChatGPT Atlas browser memories are explicitly **opt-in** ("If you opt in to Browser memories…") — [Atlas data controls](https://help.openai.com/en/articles/12574142-chatgpt-atlas-data-controls-and-privacy) (Oct 2025).

**Scope-visible memory is real, with product-specific labels.**
- ChatGPT Projects: choose "project-only" vs "default" memory at project creation; project-only chats can't reference outside conversations and the setting is irreversible — [Projects help](https://help.openai.com/en/articles/10169521-using-projects-in-chatgpt), [TechRadar](https://www.techradar.com/ai-platforms-assistants/chatgpt/chatgpt-project-only-memory-is-live-and-it-might-change-how-you-work-with-ai) (2025–26).

## CONTRADICTED / STALE

**"Opt-in by design / never persists without explicit consent" as the stated norm — contradicted by every major consumer assistant, and the gap widened in June 2026.**
- ChatGPT: models "update them automatically rather than requiring users to manage them manually… ChatGPT may save those details as a memory without you needing to ask," with after-the-fact "Memory updated" notice — [Memory FAQ](https://help.openai.com/en/articles/8590148-memory-faq). The June 4, 2026 "Dreaming" update ([openai.com/index/chatgpt-memory-dreaming/](https://openai.com/index/chatgpt-memory-dreaming/), analysis: [digitalapplied](https://www.digitalapplied.com/blog/chatgpt-memory-dreaming-v3-openai-2026-guide), [TechTimes](https://www.techtimes.com/articles/317840/20260605/chatgpt-memory-dreaming-update-openai-rewrites-personalization-engine-limits-audit-trail.htm)) moves to background synthesis "without explicit 'remember this' requests" — even less consent-gated.
- Claude: "Generate memory from chat history" is **on by default**, synthesis is silent (no per-update notification; 24h cycle) — [support.claude.com](https://support.claude.com/en/articles/11817273-use-claude-s-chat-search-and-memory-to-build-on-previous-context).
- Microsoft 365 Copilot: "No action is required to turn on Copilot memory" (default on, tenant-wide), with a post-hoc "memory updated" signal — [Microsoft Learn](https://learn.microsoft.com/en-us/copilot/microsoft-365/copilot-personalization-memory), [Community Hub](https://techcommunity.microsoft.com/blog/microsoft365copilotblog/introducing-copilot-memory-a-more-productive-and-personalized-ai-for-the-way-you/4432059).
- Gemini: Personal Context default-on since Aug 13, 2025 — [blog.google](https://blog.google/products/gemini/temporary-chats-privacy-controls/).
- Required fix: reframe as a **consent spectrum** — consumer assistants: auto-persist + notify + review-after (ChatGPT, Copilot, Gemini, Claude); coding/task agents: suggest-then-approve (Cursor Memories approvals, Devin Knowledge suggestions); opt-in reserved for high-sensitivity surfaces (Atlas browser memories). "Opt-in by design" as a universal norm will not survive a fact-check.

**The discrete key-value ledger as *the* memory model — stale; both category leaders are now summary-shaped.**
- ChatGPT Dreaming (June 4, 2026) "replaces the saved-memories list as ChatGPT's standalone foundation" with a synthesized Memory Summary Page (correct/dismiss/instruct per item; legacy Saved memories remains an option) — [digitalapplied](https://www.digitalapplied.com/blog/chatgpt-memory-dreaming-v3-openai-2026-guide), [ETIH](https://www.edtechinnovationhub.com/news/openai-rolls-out-new-chatgpt-memory-system-to-keep-personalization-current).
- Claude: "a memory summary to capture all its memories in one place for you to view and edit," updated every 24h — [claude.com/blog/memory](https://claude.com/blog/memory).
- Entry-shaped memory survives in Gemini Saved info, Copilot, Cursor, Devin. Publish the ledger pattern only with an explicit "entry-shaped vs summary-shaped" fork, or it reads twelve months stale on day one.

**Per-entry expiry ("Expires 90 days / Never") — unattested anywhere; some evidence points the opposite way.**
- ChatGPT: memories persist until deleted; Dreaming handles staleness by *rewriting* ("updates existing memories as circumstances change"), not TTL.
- Copilot: "Saved memories are retained until the end-user explicitly deletes" and — notably — "Retention policies and retention labels configured in Purview… don't apply to Copilot memory. There are no admin controls to enforce retention rules specifically for Copilot memory" — [Microsoft Learn](https://learn.microsoft.com/en-us/copilot/microsoft-365/copilot-personalization-memory). So even the enterprise product the guide's retention framing implies does not have memory TTL.
- Gemini Saved info: "stored indefinitely unless deleted manually."
- Closest real numbers are back-end, not per-entry UI: Copilot chat-history details purge 30 days after disable / 7 days after source-chat deletion; Gemini temporary chats held ≤72h. Label the expiry column a **proposal**, or reframe as enterprise-retention-policy aspiration explicitly noting no shipping product exposes it.

**"Critical for regulated environments / auditors" — reality is embarrassingly opposite.** Microsoft, the most compliance-focused vendor in the panel, ships memory with: "Memory and personalization actions don't generate audit log entries in Purview," no admin restriction on what gets memorized, and no memory retention controls — [Microsoft Learn](https://learn.microsoft.com/en-us/copilot/microsoft-365/copilot-personalization-memory) (2026-05-22). Memories *are* eDiscovery-discoverable (stored in a hidden Exchange folder, item class IPM.Contact). TechTimes' Dreaming coverage even headlines that OpenAI "limits audit trail." The guide's auditor rationale is a defensible *argument for* its design precisely because products fail here — but it must be framed as a gap analysis, not an attested requirement, and should name a real hook (eDiscovery, EU AI Act/GDPR art. 5 storage limitation) if kept.

**Session-level "memory context ring" — both named analogs measure something else, and one is in flux.**
- Cursor: composer shows a context-usage percentage with a small circle; hover reveals raw tokens ("45,000 / 200,000 tokens used") — that is context-window fill, not loaded memories. It was reportedly removed from the agent window around Cursor 2.0, prompting restore requests — [forum thread, "consumption indicator… removed"](https://forum.cursor.com/t/the-consumption-indicator-of-the-context-window-appears-to-have-been-removed/139914), [feature request](https://forum.cursor.com/t/context-window-usage/139957). Cursor 3.0's changelog (Apr 2, 2026, Agents Window) doesn't mention it — [cursor.com/changelog/3-0](https://cursor.com/changelog/3-0).
- ChatGPT: "memory full" is a *storage capacity* state (~1,200–1,400 words), now mitigated by automatic memory management (Plus/Pro) that prioritizes/deprioritizes memories — [LLMnesia](https://www.llmnesia.com/blog/chatgpt-memory-full-what-to-do), [Release Notes](https://help.openai.com/en/articles/6825453-chatgpt-release-notes) — not per-session load.
- No product shows "which memories are loaded into this session" ambiently. ChatGPT's per-response Sources is the shipping alternative. Present the ring as a proposal and explicitly disambiguate the three quantities (session-loaded memories vs. total capacity vs. context-window fill) — the current draft conflates them.

**Four content-category privacy toggles — no product segments memory by content category.** Shipping toggles are by *mechanism/tier*: ChatGPT "Reference saved memories" / "Reference chat history"; Claude "Search and reference chats" / "Generate memory from chat history"; Copilot Custom instructions / Saved memories / Chat history (and admins "can't restrict what type of information is added to memory"); Gemini Personal context / Saved info. Sources: [Memory FAQ](https://help.openai.com/en/articles/8590148-memory-faq), [support.claude.com](https://support.claude.com/en/articles/11817273-use-claude-s-chat-search-and-memory-to-build-on-previous-context), [Microsoft Learn](https://learn.microsoft.com/en-us/copilot/microsoft-365/copilot-personalization-memory). Also: the "All Off preset" framing is stale; the field's dominant all-off mechanism is a *mode* — ChatGPT Temporary Chat, Claude incognito (ghost icon, all users), Gemini Temporary Chat — not a settings preset.

- **Usage-time provenance ("Sources" on responses).** ChatGPT's May 2026 Memory Sources and Claude's chat citations put provenance at the *moment of use*, with inline correct/delete/mark-not-relevant actions — arguably the field's actual answer to the trust problem the guide's ledger-provenance addresses. The guide covers panel-side provenance only; it should cover both ends.
- **Summary-shaped memory + the supervision shift.** Dreaming makes memory self-updating and moves audit burden to the user ("supervising it became your job" — [digitalapplied](https://www.digitalapplied.com/blog/chatgpt-memory-dreaming-v3-openai-2026-guide)). A pattern for *reviewing a synthesized summary* (diffing what changed since last review) is absent from the guide and is now the leading product's core memory UX.
- **Capacity and prioritization.** ChatGPT "memory full" + automatic memory management (prioritize/deprioritize per memory), Gemini's ~2,500-token Saved info cap, Manus's 20-entry Knowledge cap. Memory as a *finite, prioritized* resource is a real UX surface with zero coverage in the guide.
- **Edit-before-save and regenerate on suggested memories.** Devin's suggestion flow (edit the proposal, or ask the agent to regenerate it from feedback) is a materially better consent interaction than binary Save/Dismiss — [docs.devin.ai](https://docs.devin.ai/product-guides/knowledge).
- **Retrieval hints as entry schema.** Devin Knowledge entries carry a "Trigger Description" ("will help Devin recall relevant Knowledge at the right times") plus optional `!macro` — a third field class beyond the guide's key/value/metadata.
- **Browser memory as a distinct memory type.** ChatGPT Atlas: opt-in browser memories with per-site visibility toggle in the address bar, archive (not just delete), search, and deletion coupled to web-history time-range deletion — [Atlas data controls](https://help.openai.com/en/articles/12574142-chatgpt-atlas-data-controls-and-privacy).
- **Enterprise governance beneath the UI.** Copilot memories live in a hidden Exchange folder (IPM.Contact/CopilotMemory), are eDiscovery-searchable/deletable, sit outside Purview retention, and generate no audit logs; Claude Enterprise offers org-level memory disable, and incognito chats are still export-retained ≥30 days for org owners ([VentureBeat](https://venturebeat.com/ai/anthropic-adds-memory-to-claude-team-and-enterprise-incognito-for-all)). The guide's Governance section has nothing on the admin/compliance plane.
- **Irreversible scope narrowing.** ChatGPT's project-only memory cannot be changed after creation — a deliberate one-way door as a privacy guarantee; a design idea worth a spec row.
- **Save-threshold policy.** Copilot's documented heuristic — persist only on "clear intent to remember" ("I prefer Python…" saved; "Write Python code…" not) — is a middle path between auto-persist and consent prompts that the guide's binary framing misses.
