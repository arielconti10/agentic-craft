## Quality read

The section is a well-crafted but almost entirely invented pattern catalog: nine plausible patterns, one fictional compliance scenario, interactive demos, and not a single grounded product observation or citation — which is conspicuous because other sections of the same guide do name real products. The single strongest idea is the **effective-policy preview** (settings toggles resolving into a live, statused readout of what the agent may/must/can't do, with cross-setting dependencies like memory being blocked until review is enabled) — it's backed by a real reusable component and is genuinely distinctive; the weakest are the numeric per-source confidence bars and the phantom 6-level autonomy scale. To be defensible publicly, the section needs product grounding passes (per the verification list) far more than it needs more prose.

**2. "Declined actions can be re-approved later from the activity log" — no product does this; eve explicitly forbids it.**

Answer an approval against a stale handle and it's rejected, so there's no way to double-resume the same parked turn." (`node_modules/eve/docs/tutorial/guard-the-spend.mdx:51-53`).
- Claude Code: a denial feeds back to the model, which continues; `/permissions` shows "recent denials" for **rule-making**, not for retroactively executing the declined action (https://code.claude.com/docs/en/permissions).

(Closest: VS Code Copilot's "resubmit requests" lets you re-send a request, which re-runs the loop — not a log-based re-approval.)
