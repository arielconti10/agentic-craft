## Staleness risks

- **Feedback-UI drift toward implicit signals.** Written March–June 2026, the area centers explicit thumbs/scales. The current direction in coding agents (Cursor Memories, Devin Knowledge) is learning from *corrective actions* (user edits, rejected diffs, follow-up messages) rather than explicit ratings; a Cursor-3-era reader may find pattern 1 and especially pattern 3 dated as primary mechanisms.
- **"Excluded from training" framing** assumes a straightforward rating→training pipeline; provider data-usage policies and post-2025 fine-tuning practice changed enough that presenting this as a spec table will age badly and was never product-observed.
- **Missing memory-editing convention.** Pattern 4's annotation strongly parallels ChatGPT's "Memory updated" chip and Cursor Memories provenance, which have matured since writing; the section should be checked against those rather than remain unattributed, or it will look like a parallel invention of a now-standard pattern.
- Demo timestamps (2026-03-14) and the landing-page promise of error reports/escalation that were never written are internal-consistency staleness, not product staleness — but they will be noticed.

## Quality read

The strongest and most distinctive idea here is **Behavioral Consequence** — showing the feedback loop closing with a traceable link from changed behavior back to the originating feedback entry; that is genuinely underexplored in shipping products and connects to real conventions (memory chips, Cursor Memories) that could ground it. The other four sections are competent but generic pattern-catalog material (thumbs, textarea-on-downvote, 1–5 scale, history list) whose demos are polished (good a11y, reduced-motion handling) but whose prose invents system consequences and overpromising microcopy with zero product grounding. Defensible after a citation pass — the repo's own `docs/research.md` already contains the HAX/HIG/PAIR/Antigravity material needed — but not publishable as-is, and the promised error-report and escalation patterns are simply missing.

## CONTRADICTED / STALE

1. **"Correction recorded — the agent will apply this in future responses" (pattern 1 confirmation copy)** — Contradicted, and the most dangerous line in the area. ChatGPT explicitly does *not* apply thumbs feedback to the current or future conversations; it "goes into a dataset used for training and evaluation cycles" ([TransferLLM](https://transferllm.com/blog/where-is-the-option-on-chatgpt-to-give-feedback-complete-guide-2026/), [OpenAI community thread](https://community.openai.com/t/whats-the-purpose-of-thumbs-up-down-feedback/105801)). Claude.ai thumbs feedback shares the conversation for training/research, with no behavior-change promise ([Claude help](https://support.claude.com/en/articles/8525154-claude-is-providing-incorrect-or-misleading-responses-what-s-going-on)). Gemini Code Assist feedback is "used only to improve" the product ([Google docs](https://developers.google.com/gemini-code-assist/docs/feedback)). The only product surface that honestly makes this promise is a **memory write** ("Memory updated" chip / Devin knowledge suggestion) — a different mechanism the guide doesn't name. Publishing this copy as-is prescribes lying to users.

2. **Textarea-only correction flow on thumbs-down** — Stale/contradicted as the default. ChatGPT, Gemini, and Copilot all lead with **reason-category chips/pickers**, with free text optional (ChatGPT) or secondary; Copilot extensions *require* a reason selection. Sources in CONFIRMED #1. The guide should present chips+optional-text as the observed default and textarea-first as a deliberate deviation.

3. **1–5 in-thread rating with per-value system consequences (pattern 3)** — Contradicted as an end-user pattern. No panel product (ChatGPT, Claude.ai, Gemini, Copilot, Cursor, Devin, v0, Perplexity) ships an in-thread 5-point scale; binary thumbs dominates, and 1–5 scales appear only in **post-session CSAT surveys** and internal eval/labeling tooling ([Quickchat CSAT guide](https://quickchat.ai/post/chatbot-csat-score-guide), [Nurix chatbot metrics](https://www.nurix.ai/blogs/essential-chatbot-evaluation-metrics-success)). The consequence table (1 = "excluded from training," 2 = "queued for regeneration") is invented backend behavior with no product analogue and conflicts with how providers actually describe feedback pipelines (offline, batched, on the provider's schedule). Needs reframing as an internal-reviewer/labeling pattern or cutting.

4. **Explicit-rating-centric framing of the whole area** — Stale drift confirmed. The 2026 direction in coding agents is implicit learning from corrective actions: Devin auto-suggests knowledge "based on your feedback in chat" ([Devin docs](https://docs.devin.ai/product-guides/knowledge)), Cursor's background model proposes memories from conversations, and Claude Code's auto memory "lets Claude learn from your corrections without manual effort" ([code.claude.com/docs/en/memory](https://code.claude.com/docs/en/memory)). Thumbs/scales are now the *secondary* channel in agentic tools; the primary loop is correction → proposed persistent entry → user approval/edit.

5. **"Cursor Memories" as a citation target** — Stale naming. `docs.cursor.com/context/memories` now 308-redirects to `cursor.com/docs`, whose context docs cover Rules (project/user/team/AGENTS.md) with no Memories page; current changelog language is "memory files" (deletable in UI since 3.7, June 17, 2026) and positions Skills as complementary to Rules ([cursor.com/changelog](https://cursor.com/changelog), fetched 2026-07-02). Cite Cursor with current terminology or the citation itself will read as stale.
